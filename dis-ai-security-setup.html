Codex & Claude Code 從零設定指南

公司網路有管制、公司資料絕對不能外洩。這份指南帶同仁從安裝那一刻起，把 Codex 與 Claude Code 的沙盒、核准政策、權限與隱私一項一項設好——預設關閉網路、只在工作資料夾內動作、每一步都要你點頭，讓你在工作上能安心用 AI。

00// WHY

為什麼要先做這份設定

AI Agent 很強，是因為它能「動手」——而能動手，就代表它有機會把資料送出去。

Codex 和 Claude Code 這類 AI Agent，跟一般聊天 AI 不一樣：它能讀你的檔案、執行命令、甚至連上網路。用對了是超強的助理；但若沒設定好，它可能在你沒注意時，把公司的程式碼、客戶名單、報價、測試資料傳到外部。

我們最怕的一件事：資料外洩

公司網路本來就有管制。我們的目標是：就算同仁用了 AI，公司資料也出不去。做法是把工具的「對外能力」預設關掉，需要時再由人逐次核准。這份指南就是要你從全新安裝、從 0 開始，照著把每一個開關設到安全的位置。

預設最安全

裝好後第一件事，就是把設定調到「最保守」，而不是貪方便全開。

先問再做

任何越界動作（連網、出資料夾）都要先問你，你看過才放行。

關起來用

沙盒 + 關網路，讓 AI 只能在它的小空間裡做事，碰不到外面。

01// PRINCIPLES

六大核心原則

兩個工具設定細節不同，但背後都是這六條。記住原則，設定就不會錯。

① 預設關閉網路

沒有對外網路 = 資料送不出去。這是最重要的一條。需要查網路時，再「逐次」開、用完關。

② 只在工作資料夾內

把 AI 限制在「這個專案資料夾」內，不要讓它能讀寫整台電腦、家目錄或憑證檔。

③ 每次越界都要核准

核准政策設成「依請求」。執行命令、出資料夾、連網路前，AI 都要先問、你才放行。

④ 永不開「完整存取」

「完整存取 / danger-full-access / --yolo」會拿掉所有防護。公司機器一律禁用。

⑤ 不亂接外掛與 MCP

不認識的 MCP 伺服器、瀏覽器外掛不要接。它們可能被植入指令、把資料帶走。

⑥ 關遙測、不貼機密

關掉遙測與自動回報；機密資料不要貼進公開的雲端聊天視窗。

一句話記住

「關網路、留在資料夾、先問再做、不開全權。」做到這四件事，外洩風險就被壓到很低。

02// THREAT MAP

資料外洩風險地圖

資料可能從哪裡漏出去？我們又是用哪個設定擋住它？

外洩途徑會怎麼漏我們怎麼擋
連網命令curl / wget 把檔案上傳到外部網站沙盒關網路 + 拒絕這些命令
內建上網工具AI 用 WebFetch / 瀏覽器把內容送出預設拒絕 WebFetch / WebSearch，要用再逐次開
遠端傳檔scp / ssh / rsync 傳到外部主機沙盒擋 + 拒絕這些命令
推到雲端 Gitgit push 把程式碼推到外部 repo關網路 + 對 push 要求核准
外掛 / MCP不明 MCP 伺服器把資料帶走，或植入惡意指令不接不明 MCP；預設封鎖
憑證外流讀到 .env、SSH 金鑰、API Key 後送出限制只在工作資料夾；不讀家目錄與憑證
遙測 / 訓練使用紀錄、提示詞被上傳或用於訓練關遙測；用商用方案、確認不拿你的資料訓練
核心觀念：縱深防禦

不靠單一開關。我們同時用「沙盒（關網路）＋ 權限規則（拒絕危險命令）＋ 核准（人來把關）」三層，任何一層擋住就安全。

03// CODEX · 安裝

安裝 Codex

不論你用的是 Codex App 還是命令列（CLI），設定的觀念都一樣。

A

Codex App（圖形介面）

  1. 從公司核准的來源安裝 Codex App。
  2. 用公司核准的帳號 / 方案登入（不要用個人免費帳號處理公司資料）。
  3. 登入後先別急著用，直接進「設定」把第 04、05 章的開關設好。

設定路徑：左側選單 →「組態 / 一般」。

B

Codex CLI（命令列）

  1. 用 Node.js 安裝（需公司核准的網路 / 鏡像）：
npm install -g @openai/codex
  1. 第一次執行會建立設定檔：
~/.codex/config.toml

CLI 的安全設定全部寫在這個檔案，見第 06 章範本。

裝好後的第一步永遠是「設定」

不要裝完就直接丟公司資料進去用。先把沙盒、核准、完整存取這幾個開關設到安全位置（下兩章），再開始工作。

04// CODEX · 一般

一般設定建議

對照 Codex App「一般」頁面，一項一項照下表設。

設定項（App 中文）建議值為什麼
工作模式「用於程式設計」或「適合日常工作」皆可只影響顯示多少技術細節，不影響安全。新手選「適合日常工作」。
預設權限開（可讀寫工作區）讓 AI 能在「工作資料夾」內讀寫檔案是正常需要，安全。
自動審查關這會讓 AI「自動同意」額外存取要求。關掉，改由你來核准。
完整存取權關（務必）開了等於可編輯整台電腦的任何檔案、未經同意就連網。風險最高，禁止。
代理程式環境依公司規範（Windows 原生 / WSL）不影響外洩風險；統一用公司指定的即可。
整合的終端 ShellPowerShell（或公司指定）習慣問題，不影響安全。
Codex 相依項目視需要允許安裝隨附的 Node.js / Python 工具。屬本機安裝，需要才開。
三個開關的重點

預設權限＝開（只在工作區）、自動審查＝關、完整存取權＝關。記住這組合就對了。

05// CODEX · 最重要

沙盒與核准政策

這一頁是整份指南最關鍵的地方——設好它，資料基本就出不去。

沙盒設定（Sandbox）

沙盒決定「Codex 執行命令時能做到什麼」。三個選項：

沙盒模式能做什麼建議
唯讀 read-only只能看檔案，不能改、不能執行命令（要做都要核准）最安全 · 預設首選
工作區可寫 workspace-write可讀、可改「工作資料夾」內的檔、可跑本機命令；網路預設關閉需要跑程式時才用
完整存取 danger-full-access沒有任何限制：可碰整台電腦、可連網禁止使用
關鍵：workspace-write 預設「不能連網」

就算選了「工作區可寫」，Codex 預設也不能上網。除非有人特地把網路打開，否則資料無法被送出去。所以平時請不要打開網路存取。

核准政策（Approval policy）

核准政策意思建議
依請求 on-request預設在沙盒內工作，要越界（連網、出資料夾）時先問你建議
untrusted只要不是受信任的命令，都先問更嚴格亦可
never 從不詢問完全不停下來問，全自動執行禁止使用

沙盒設定 → 選「唯讀」

一般文件處理、看資料、問問題，唯讀就夠。需要它實際跑程式 / 改檔，再臨時切「工作區可寫」。

核准政策 → 選「依請求」

任何越界動作都會跳出來問你，看清楚再按同意。

網路存取 → 保持關閉

workspace-write 下不要打開 network access。真的需要查網路，當次開、用完關。

完整存取 → 永遠關

不論 App 或 CLI，都不要啟用 danger-full-access。

06// CODEX · CLI

config.toml 安全範本

用 CLI 的同仁，把這段貼進 ~/.codex/config.toml 即可。

~/.codex/config.toml
# ===== DIS Tech 安全預設 =====
# 越界(連網/出工作區)前一律先問你
approval_policy = "on-request"

# 預設唯讀最安全；要實際跑程式再改成 "workspace-write"
sandbox_mode = "read-only"

# 即使切到 workspace-write，也務必保持網路關閉
[sandbox_workspace_write]
network_access = false   # ← 關鍵：關網路 = 資料出不去

# 隱私：不要把你的提示詞(可能含原始碼)寫進記錄
log_user_prompt = false
臨時要跑程式時

把 sandbox_mode 改成 "workspace-write" 即可——它仍只能在工作資料夾內動作、且網路仍是關的。工作完可以再改回 "read-only"。

07// CODEX · 紅線

Codex 絕對不要做

以下任一個，等於把所有防護拆掉。

不要開「完整存取權 / danger-full-access」

它會移除檔案與網路的所有界線，AI 可在未經你同意下連網、改任何檔。公司機器禁止。

不要用 --dangerously-bypass-approvals-and-sandbox（別名 --yolo）

CLI 的「無沙盒、無核准」模式。沒有任何把關，最危險。

不要把核准政策設成「never」

等於全自動放行，AI 做什麼都不問你。

安裝 Claude Code

  2. 使用者設定檔位置：

09// CLAUDE CODE · 權限

權限模式與沙盒

三件事：起始模式設保守、拒絕外洩工具、（進階）開沙盒。

起始權限模式（defaultMode）

模式意思建議
plan 計畫唯讀探索，不會改檔；先看它的計畫再放行最安全 · 起始首選
default 預設第一次用到某工具時會先問你日常可用
acceptEdits自動同意檔案編輯熟悉後、在工作區內才用
bypassPermissions跳過所有詢問禁止使用

拒絕會外洩的工具（最重要）

用 permissions.deny 把「能把資料送出去」的工具與命令直接封鎖。被 deny 的規則最優先、不能被例外放行。

沙盒（進階）

Claude Code 另有 OS 層沙盒，可限制 Bash 命令的檔案與網路存取。Windows 需在 WSL2 下使用。設定鍵與細節以官方文件為準：code.claude.com/docs/en/sandboxing。沒有 WSL 也沒關係——靠下面的 deny 規則＋plan 模式就已足夠安全。

10// CLAUDE CODE · 範本

settings.json 安全範本

貼進 ~/.claude/settings.json。這份預設「不上網、先問再做、不接 MCP」。

~/.claude/settings.json
{
  "permissions": {
    // 起始用最保守的 plan(唯讀計畫)模式
    "defaultMode": "plan",

    // 封鎖所有「把資料送出去」的工具與命令
    "deny": [
      "WebFetch",
      "WebSearch",
      "Bash(curl:*)",
      "Bash(wget:*)",
      "Bash(scp:*)",
      "Bash(ssh:*)",
      "Bash(rsync:*)",
      "Bash(git push:*)",
      "mcp__*"            // 預設封鎖所有 MCP 外掛
    ],

    // 高風險命令一律先問
    "ask": [
      "Bash(rm:*)",
      "Bash(sudo:*)"
    ]
  },

  // 關閉遙測、錯誤回報、自動更新與非必要連線
  "env": {
    "DISABLE_TELEMETRY": "1",
    "DISABLE_ERROR_REPORTING": "1",
    "DISABLE_AUTOUPDATER": "1",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1"
  }
}
需要某個被擋的功能時

例如真的要查網路，請逐次用 /permissions 臨時允許特定網域，用完再拿掉；不要直接把整條 deny 刪掉。

命令規則寫法

Bash(curl:*) 代表「任何 curl 開頭的命令」。你也可以加 allow 白名單（如 Bash(npm run test:*)）只放行特定安全命令。完整語法見 官方權限文件。

11// CLAUDE CODE · 隱私

隱私與遙測

把這些環境變數設成關閉，減少對外的非必要連線。

環境變數設成作用
DISABLE_TELEMETRY1關閉使用情形遙測
DISABLE_ERROR_REPORTING1關閉錯誤回報上傳
DISABLE_AUTOUPDATER1關閉自動更新（改由 IT 控管版本）
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC1一鍵關閉各種非必要對外連線
關於「不拿你的資料訓練」

是否用你的資料訓練，取決於帳號方案：商用方案（Team / Enterprise / Console / API）受商用條款保護、預設不拿來訓練；個人 Free/Pro 則要到帳號隱私設定關閉。處理公司資料請一律使用公司核准的商用方案。詳見 Anthropic 隱私政策。

12// CLAUDE CODE · 紅線

Claude Code 絕對不要做

這些等於關掉所有防護。

不要用 --dangerously-skip-permissions（＝ bypassPermissions 模式）

會跳過所有權限詢問。只有在「完全隔離、丟掉也沒差」的容器 / 虛擬機才考慮，公司日常機器禁止。

不要隨便把整段 deny 刪掉

需要某功能就逐次開、用完關，不要為了方便把網路 / curl / git push 的封鎖整個拿掉。

不要接不明的 MCP 伺服器

MCP 回傳的內容可能藏有惡意指令（提示注入），誘導 AI 把資料送出去。只接公司核可的來源。

13// FOR IT

給 IT 的部署清單

想全公司統一鎖定、不讓同仁自行放寬，用「強制設定」最有效。

  • Codex：用 requirements.toml 強制禁止 approval_policy = "never" 與 sandbox_mode = "danger-full-access"。
  • Claude Code：部署 managed settings 到 C:\Program Files\ClaudeCode\managed-settings.json（使用者無法覆蓋）。
  • 在 managed settings 內放入第 10 章的 deny 規則與遙測 env，並停用 bypassPermissions 模式。
  • 預設封鎖 MCP（"mcp__*" deny），只開放公司白名單。
  • 統一登入公司核准的商用方案（確保不拿資料訓練、可稽核）。
  • 搭配公司既有的網路出口管制 / Proxy，做雙重保險。
  • 教同仁用 /permissions（Claude Code）檢視目前生效的規則。
強制設定 vs. 使用者設定

使用者自己設的可被改回去；IT 強制層（managed / requirements）使用者無法覆蓋，是最可靠的防線。一般同仁照 03–12 章設好即可，IT 再加這層強制。

14// HUMAN RULES

員工使用守則

工具設好之外，人也要養成習慣。

✓ 請這樣做

  • 用公司核准的帳號 / 商用方案。
  • 在專屬的工作資料夾內開 AI，不要在桌面 / 整顆 C 槽跑。
  • 跳出核准視窗時看清楚再按同意。
  • 要連網 / 推 Git 時逐次開、用完關。
  • 定期檢視生效的權限規則。

✗ 請不要

  • 把客戶名單、報價、原始碼、機密貼進公開的雲端聊天。
  • 開完整存取 / --yolo / bypass 等全權模式。
  • 為了省事把網路與封鎖規則整個關掉。
  • 接來路不明的 MCP / 外掛 / 連結。
  • 用個人免費帳號處理公司資料。
